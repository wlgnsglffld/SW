클래스는 총 5개로 메인메뉴 관리자메뉴 사용자메뉴 데이터베이스를 연결하는 DAO 학생 정보 테이블 데이터 표현을 위한 study클래스 이렇게 구성하였습니다 
데이터베이스에 dbstudy라는 데이터베이스를 만들고 study와 incorr이라는 테이블을 2개 만들었습니다.
study 테이블은 단어장의 설명과 단어를 저장하고
incorr 테이블은 단어시험의 정답을 체크하도록 하였습니다.

메인 메뉴는 2가지로 사용자와 관리자 입니다.
먼저 여기서 설명드릴점은 미리 저희가 퀴즈모드를 사용하여 틀린문제가 저장되게하였습니다
(select * from incorr;) 이 이유는 메인 메뉴에서 사용자 버튼을 누르면 오답을 넣은 incorr테이블을 초기화 하도록 resetIncorr 메소드를 넣어두었습니다. resetIncorr 메소드는 delete from incorr을 이용하여 사용자가 퀴즈모드에 이용을 용이하도록 하였습니다.
(CMD에 저장되어있는 오답 모습 보여주기)

먼저 관리자 모드 실행결과와 코드에 대해 설명하겠습니다.
관리자모드에서 단어추가는 boolean insertStudy 메소드를 이용하여 데이터 베이스에 문제를 preparedStatement에 파라미터 설정하여 추가하도록 하였습니다. 
(추가5개되는 모습 보여주기)


다음으로는 study 테이블에 저장되어있는 단어를 삭제하는 deletStudy 메소드입니다
위와 마찬가지로 preparedStatement에 파라미터 설정하여 삭제하도록 하였습니다
여기서 코드에 보시면  preparedStatement ("SET @num := 0") 이라는 변수를 만들어 UPDATE study SET number = @num := @num+1" 이라는 실행문을 사용해 테이블에있는 연번을 초기화 하여 재배치합니다
 (16번까지 문제가 있다는 가정으로 14번을 삭제해서 앞으로 당져지는거 보여주기)

수정은 updateStudy 메소드를 사용하여 테이블에 저장되어있는 단어의 설명을 수정하는것입니다. 수정시 if문을 사용하여 테이블에 저장되어있는 단어가 아닌경우 일치하는 값이 없어 검색이 되지않도록 하였습니다. 저장되어있는 단어가 있을경우 if 문을 이용하여 detail.setText(Question)으로 설명을 불러오고 (Sdao.updateStudy(study))로 문제 정보를 업데이트하였습니다 ******else를 물어보면 varchar 값 초과*******

다음으로 검색모드인데 이 기능은 사용자와 관리자 모두 해당되는 기능입니다.
searchStudy를 이용하여 String으로 문자열을 입력받아 데이터베이스에 저장되어있는 값을 찾아보는 기능입니다. 이 또한 마찬가지로 if 문을 이용하여 데이터베이스에 저장된 단어가 아닌 단어를 입력하면 일치하는 단어가 없다고 나오도록 하였습니다.
